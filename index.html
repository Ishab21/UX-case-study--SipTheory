Sip Theory

A calm, sensory-first boba experience.

Sip Theory reimagines boba ordering as a slow, immersive ritual. Instead of clutter and urgency, the design reduces cognitive load and encourages intentional browsing.

Project Overview

The problem

Most boba websites feel cluttered, high-energy, and transactional. Sip Theory was designed to reduce cognitive overload and create a smoother, more premium browsing experience.

Problem

Most boba sites are overwhelming, bright, and transactional.

Insight

The interface should reflect the calm and sensory feel of the drink itself.

Solution

A slow, vertical storytelling layout instead of a dense grid-based structure.

Goal

Reduce stress and create a more intentional, premium browsing journey.

Layout Thinking

Why this layout?

I chose a vertical scrolling structure to slow users down. Traditional grids encourage scanning and rushing, while a story-based flow creates a more immersive and emotional experience.

“Browsing should feel like sipping slowly.”

UX Features

What I added for a smoother browsing experience

Each feature was designed to reduce cognitive overload and make browsing feel calmer, clearer, and more intentional.

1. Story-based scrolling

I used a layered vertical flow so users move through the brand in sequence instead of rushing through crowded information.

2. Minimal navigation

The navigation stays limited and clear so users can orient themselves quickly without too many competing options.

3. Editorial menu layout

The menu is presented like a curated story instead of a dense product grid, making the drinks feel more premium and memorable.

4. Clean location system

The location page uses strong hierarchy through the map and store cards so users can scan information more easily.

5. Low-friction contact

The contact section keeps interaction simple, approachable, and visually consistent with the rest of the site.
